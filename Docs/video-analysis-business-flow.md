# 视频解析功能业务流程设计

## 📋 业务概述

视频解析功能是一个基于AI的智能视频内容分析系统，用户可以选择已上传的视频文件，配置解析参数（提示词模板、标签组、AI配置），然后通过AI API对视频进行深度分析，最终获得结构化的Markdown格式解析结果。

## 🎯 核心价值

- **智能化分析**：利用AI技术自动分析视频内容、技术指标和营销价值
- **模板化配置**：预设多种分析模板，满足不同场景需求
- **标签化管理**：通过标签组系统实现精准的内容分类和特征标注
- **实时反馈**：流式返回分析结果，提供良好的用户体验
- **结果可视化**：Markdown格式输出，支持富文本展示和导出

## 🔄 完整业务流程

### 阶段1：准备阶段

#### 1.1 视频文件准备
- **前置条件**：用户已通过视频上传功能上传了视频文件
- **数据来源**：`uploaded_files` 表中的视频记录
- **文件状态**：确保视频文件完整上传且元数据提取完成
- **可选视频**：系统展示最近上传的3个视频供用户选择

#### 1.2 配置资源准备
- **提示词模板**：系统预设多种分析模板（内容分析、技术评估、营销价值等）
- **标签组系统**：预定义的标签分类（视频类型、技术特征、内容风格等）
- **AI配置**：管理员预配置的AI服务（OpenAI、Claude等）
- **权限验证**：确保用户有权限访问解析功能

### 阶段2：配置阶段

#### 2.1 视频选择
```
用户操作：
1. 进入视频解析页面
2. 查看最近上传的视频列表
3. 选择要分析的目标视频

系统处理：
1. 调用 GET /api/v1/video-analysis/videos/recent
2. 返回视频基本信息（文件名、大小、时长、分辨率等）
3. 前端展示视频缩略图和元数据
```

#### 2.2 模板选择
```
用户操作：
1. 浏览可用的提示词模板
2. 查看模板内容预览
3. 选择适合的分析模板

系统处理：
1. 调用 GET /api/v1/video-analysis/templates
2. 返回活跃模板列表
3. 显示模板标题、内容摘要和使用次数
```

#### 2.3 标签组配置
```
用户操作：
1. 查看可用标签组
2. 选择相关的标签分类
3. 可选择多个标签组进行组合

系统处理：
1. 调用 GET /api/v1/video-analysis/tag-groups
2. 返回标签组及其包含的标签
3. 支持多选和预览功能
```

#### 2.4 AI配置选择
```
用户操作：
1. 选择AI服务提供商和模型
2. 查看模型能力和限制
3. 确认配置参数

系统处理：
1. 调用 GET /api/v1/video-analysis/ai-configs
2. 返回可用AI配置（不含敏感信息）
3. 显示提供商、模型、参数等信息
```

#### 2.5 自定义提示词（可选）
```
用户操作：
1. 可以覆盖或补充模板内容
2. 添加特定的分析要求
3. 预览最终提示词内容

系统处理：
1. 合并模板内容和自定义内容
2. 整合选中的标签信息
3. 生成最终的提示词
```

### 阶段3：执行阶段

#### 3.1 解析任务创建
```
用户操作：
1. 确认所有配置参数
2. 点击"开始解析"按钮
3. 等待任务创建确认

系统处理：
1. 验证所有输入参数
2. 检查视频文件和AI配置可用性
3. 创建VideoAnalysis记录
4. 返回任务ID和流式URL
```

#### 3.2 提示词生成
```
系统内部处理：
1. 获取选中的提示词模板内容
2. 收集所有选中标签组的标签
3. 合并自定义提示词（如有）
4. 生成最终的完整提示词
5. 存储到analysis记录中

提示词结构示例：
[模板内容]

相关标签: 教育, 高清, 正式, 专业

[自定义补充内容]
```

#### 3.3 后台异步处理
```
系统后台任务：
1. 更新任务状态为"processing"
2. 获取AI配置和视频文件信息
3. 构建AI API请求
4. 调用AI服务进行分析
5. 处理AI响应结果
6. 更新进度和结果
```

### 阶段4：监控阶段

#### 4.1 实时进度监控
```
用户体验：
1. 页面显示解析进度条
2. 实时更新状态信息
3. 显示预估剩余时间

技术实现：
1. 建立EventSource连接
2. 接收Server-Sent Events
3. 更新前端UI状态
4. 处理错误和异常情况
```

#### 4.2 流式结果接收
```
数据流处理：
1. 接收进度更新事件
2. 接收部分内容事件
3. 接收完成或错误事件
4. 实时渲染Markdown内容

事件类型：
- progress: 进度更新
- content: 内容片段
- error: 错误信息
- complete: 完成通知
```

### 阶段5：结果阶段

#### 5.1 结果展示
```
用户界面：
1. Markdown格式的分析结果
2. 语法高亮和富文本渲染
3. 结果质量评分显示
4. 处理时间和成本信息

结果结构：
- 基本信息（文件名、时长、分辨率等）
- 内容分析（视觉、音频、风格等）
- 技术指标（编码、质量、兼容性等）
- 总结建议（优化建议、适用场景等）
```

#### 5.2 结果操作
```
用户可执行操作：
1. 复制结果内容
2. 导出为文件
3. 分享结果链接
4. 重新分析（使用不同配置）
5. 查看历史记录
```

#### 5.3 历史记录管理
```
系统功能：
1. 保存所有解析记录
2. 支持按状态筛选
3. 分页浏览历史
4. 快速重新查看结果
5. 删除不需要的记录
```

## 📊 数据流转图

```
用户选择视频 → 配置解析参数 → 提交解析请求
     ↓                ↓              ↓
视频文件信息 ← 模板+标签+AI配置 → 创建解析任务
     ↓                              ↓
后台异步处理 ← 生成完整提示词 ← 调用AI API
     ↓                              ↓
实时进度更新 → 流式结果返回 → 最终结果展示
     ↓                              ↓
历史记录保存 ← 用户操作结果 ← 结果质量评估
```

## 🔧 关键业务规则

### 数据验证规则
1. **视频文件验证**：确保文件存在且完整
2. **模板验证**：检查模板是否激活和可用
3. **标签组验证**：验证标签组和标签的有效性
4. **AI配置验证**：确认AI服务可用且有权限
5. **并发控制**：同一视频同时只能有一个解析任务

### 业务约束
1. **文件大小限制**：根据AI服务限制设定最大文件大小
2. **并发任务限制**：控制同时进行的解析任务数量
3. **结果保留期**：设定解析结果的保存时间
4. **权限控制**：确保用户只能访问自己的解析记录
5. **成本控制**：监控AI API调用成本和频率

### 异常处理
1. **网络异常**：AI API调用失败的重试机制
2. **超时处理**：长时间无响应的任务超时处理
3. **资源不足**：系统资源不足时的降级策略
4. **数据异常**：无效数据的错误提示和恢复
5. **用户中断**：用户主动取消任务的处理

## 🎨 用户体验设计

### 界面交互流程
1. **引导式配置**：步骤化的配置向导
2. **实时预览**：配置过程中的实时预览
3. **智能推荐**：基于视频特征的配置推荐
4. **快速重试**：一键使用不同配置重新分析
5. **结果对比**：多次分析结果的对比功能

### 响应式设计
1. **移动端适配**：支持手机和平板设备
2. **加载状态**：优雅的加载动画和进度指示
3. **错误提示**：友好的错误信息和解决建议
4. **性能优化**：大文件和长结果的分页加载
5. **离线支持**：结果缓存和离线查看功能

## 📈 性能和监控

### 性能指标
1. **响应时间**：API接口的平均响应时间
2. **处理时长**：视频解析的平均处理时间
3. **成功率**：解析任务的成功完成率
4. **并发能力**：系统支持的最大并发任务数
5. **资源利用率**：CPU、内存和存储的使用情况

### 监控告警
1. **任务失败率**：解析失败率超过阈值时告警
2. **响应时间**：API响应时间过长时告警
3. **资源使用**：系统资源使用率过高时告警
4. **AI API状态**：外部AI服务异常时告警
5. **存储空间**：结果存储空间不足时告警

## 🔒 安全和隐私

### 数据安全
1. **API密钥保护**：AI服务密钥不暴露给前端
2. **文件访问控制**：确保用户只能访问自己的文件
3. **结果加密**：敏感分析结果的加密存储
4. **审计日志**：记录所有关键操作的审计日志
5. **数据清理**：定期清理过期的临时数据

### 隐私保护
1. **数据最小化**：只收集必要的用户数据
2. **匿名化处理**：对分析数据进行匿名化
3. **用户同意**：明确的数据使用授权
4. **数据删除**：用户可以删除自己的分析记录
5. **第三方隔离**：确保AI服务不保留用户数据

**业务流程总结**：

这是一个完整的端到端视频解析业务流程，从用户选择视频开始，经过配置、执行、监控到最终结果展示的全过程。系统通过模板化和标签化的方式，让用户能够灵活配置分析参数，通过AI技术实现智能化的视频内容分析，并以流式的方式提供实时反馈和结果展示。整个流程注重用户体验、系统性能和数据安全，为用户提供专业、高效、安全的视频分析服务。
