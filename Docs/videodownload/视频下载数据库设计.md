# 视频下载功能数据库设计

## 1. 设计概述

### 1.1 设计原则
1. **数据一致性**：与现有视频上传模块保持数据结构一致性
2. **扩展性**：支持新平台和功能的扩展
3. **性能优化**：合理的索引设计和查询优化
4. **数据完整性**：完善的约束和关联关系

### 1.2 现有表结构分析
- `videos`表：已包含完整的视频元数据结构
- `download_tasks`表：已有基础下载任务结构
- `download_history`表：已定义下载历史记录结构
- `users`表：用户信息和权限管理

#### 2.1.2 平台和URL字段详细说明

**platform字段 (VARCHAR(50))**
- **用途**: 记录视频来源平台
- **支持平台**: 'douyin', 'tiktok', 'bilibili', 'xiaohongshu', 'kuaishou', 'weixin', 'youtube'
- **约束**: 非空，建议使用枚举值
- **索引**: 已创建索引以优化查询性能

**original_url字段 (TEXT)**
- **用途**: 记录视频的原始URL地址
- **格式**: 完整的HTTP/HTTPS URL
- **约束**: 对于下载的视频必须非空
- **用途场景**: 
  - 视频溯源和版权管理
  - 重新下载和更新
  - 用户分享和引用
  - 数据分析和统计

#### 2.2.2 平台和URL字段详细说明

**url字段 (TEXT, NOT NULL)**
- **用途**: 记录用户提交的原始视频URL
- **格式**: 完整的HTTP/HTTPS URL
- **约束**: 必须非空，是下载任务的核心标识
- **支持格式**: 
  - 抖音: `https://v.douyin.com/xxx` 或 `https://www.douyin.com/video/xxx`
  - TikTok: `https://www.tiktok.com/@user/video/xxx` 或 `https://vm.tiktok.com/xxx`
  - B站: `https://www.bilibili.com/video/BVxxx` 或 `https://b23.tv/xxx`
  - 小红书: `https://www.xiaohongshu.com/explore/xxx`
  - 快手: `https://www.kuaishou.com/short-video/xxx`

**platform字段 (VARCHAR(50))**
- **用途**: 自动识别或手动指定的视频平台
- **识别方式**: 通过URL模式匹配自动识别
- **支持平台**: 与videos表保持一致
- **索引**: 创建复合索引 (user_id, platform, status) 优化查询

## 3. 数据关系设计

### 3.1 实体关系图
```
users (1) -----> (N) download_tasks
download_tasks (1) -----> (0..1) videos
download_tasks (1) -----> (0..1) download_queue
download_tasks (1) -----> (N) download_history
users (1) -----> (N) download_statistics
download_platforms (1) -----> (N) download_tasks (通过platform字段关联)
```

### 3.2 关键约束

#### 3.2.1 外键约束
```sql
-- 确保数据完整性的外键约束
ALTER TABLE download_tasks ADD CONSTRAINT fk_download_tasks_user_id 
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE;

ALTER TABLE download_tasks ADD CONSTRAINT fk_download_tasks_video_id 
    FOREIGN KEY (video_id) REFERENCES videos(id) ON DELETE SET NULL;

ALTER TABLE videos ADD CONSTRAINT fk_videos_download_task_id 
    FOREIGN KEY (download_task_id) REFERENCES download_tasks(id) ON DELETE SET NULL;

ALTER TABLE download_queue ADD CONSTRAINT fk_download_queue_task_id 
    FOREIGN KEY (task_id) REFERENCES download_tasks(id) ON DELETE CASCADE;

ALTER TABLE download_statistics ADD CONSTRAINT fk_download_statistics_user_id 
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE;
```

#### 3.2.2 检查约束
```sql
-- 状态值约束
ALTER TABLE download_tasks ADD CONSTRAINT chk_download_tasks_status 
    CHECK (status IN ('pending', 'analyzing', 'downloading', 'completed', 'failed', 'cancelled'));

ALTER TABLE download_queue ADD CONSTRAINT chk_download_queue_status 
    CHECK (status IN ('queued', 'processing', 'completed', 'failed'));

-- 优先级约束
ALTER TABLE download_tasks ADD CONSTRAINT chk_download_tasks_priority 
    CHECK (priority >= 1 AND priority <= 10);

ALTER TABLE download_queue ADD CONSTRAINT chk_download_queue_priority 
    CHECK (priority >= 1 AND priority <= 10);

-- 进度约束
ALTER TABLE download_tasks ADD CONSTRAINT chk_download_tasks_progress 
    CHECK (progress >= 0 AND progress <= 100);
```

## 6. 性能优化建议

### 6.1 查询优化

#### 6.1.1 分页查询优化
```sql
-- 使用游标分页替代OFFSET
SELECT * FROM download_tasks 
WHERE user_id = ? AND id > ? 
ORDER BY id 
LIMIT 20;
```

#### 6.1.2 统计查询优化
```sql
-- 使用物化视图进行统计查询优化
CREATE MATERIALIZED VIEW download_daily_stats AS
SELECT 
    DATE(created_at) as date,
    platform,
    COUNT(*) as total_downloads,
    COUNT(CASE WHEN status = 'completed' THEN 1 END) as successful_downloads,
    COUNT(CASE WHEN status = 'failed' THEN 1 END) as failed_downloads,
    SUM(file_size) as total_size
FROM download_tasks
GROUP BY DATE(created_at), platform;

-- 定期刷新物化视图
REFRESH MATERIALIZED VIEW download_daily_stats;
```

### 7.2 数据同步

#### 7.2.1 任务状态同步
```sql
-- 确保download_tasks和download_queue状态同步
CREATE OR REPLACE FUNCTION sync_download_status()
RETURNS TRIGGER AS $$
BEGIN
    IF NEW.status != OLD.status THEN
        UPDATE download_queue 
        SET status = CASE 
            WHEN NEW.status = 'completed' THEN 'completed'
            WHEN NEW.status = 'failed' THEN 'failed'
            WHEN NEW.status = 'downloading' THEN 'processing'
            ELSE 'queued'
        END,
        updated_at = CURRENT_TIMESTAMP
        WHERE task_id = NEW.id;
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_sync_download_status
    AFTER UPDATE ON download_tasks
    FOR EACH ROW
    EXECUTE FUNCTION sync_download_status();
```

## 8. 监控和维护

```sql
-- 下载成功率
SELECT 
    platform,
    COUNT(*) as total,
    COUNT(CASE WHEN status = 'completed' THEN 1 END) as success,
    ROUND(COUNT(CASE WHEN status = 'completed' THEN 1 END) * 100.0 / COUNT(*), 2) as success_rate
FROM download_tasks
WHERE created_at >= CURRENT_DATE - INTERVAL '7 days'
GROUP BY platform;

-- 平均下载时间
SELECT 
    platform,
    AVG(EXTRACT(EPOCH FROM (completed_at - started_at))) as avg_download_time
FROM download_tasks
WHERE status = 'completed' AND started_at IS NOT NULL AND completed_at IS NOT NULL
GROUP BY platform;
```
